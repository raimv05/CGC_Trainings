SPA vs MPA & the Virtual DOM
Click "Simulate Update" to see the Virtual DOM in action
Traditional Website (MPA)
Home↓ reloadAbout↓ reloadContact
React (SPA)
Home↓ component changesNo Reload
Analogy: editing one word in a Word document — you wouldn't reprint the entire document. React updates only the part of the page that changed, not the whole thing.
Virtual DOM — click to simulate a state change
Real DOM would redraw everything. Virtual DOM only touches the highlighted cell.

Day 21 · Part 1
Destructuring Props
A cleaner way to read props, without repeating `props.` everywhere
Before
function CourseCard(props){
  return <h3>{props.name}</h3>;
}
After
function CourseCard({
  name, trainer, duration
}){
  return <h3>{name}</h3>;
}

Day 21 · Part 2
State with useState
Props can't change over time by themselves — state can
The Problem
Course count is currently static. We need it to be dynamic — updating live as the user interacts.
App.jsx
import {useState} from "react";

const [count, setCount] = useState(0);
count
The current value — read it directly in JSX.
setCount
The only correct way to update it and trigger a re-render.

alert(…)
Built-in browser function
Tells the browser to show a popup dialog with any message you give it.
alert() is a method of the window object — technically window.alert() — but JavaScript lets you skip the window. prefix. It's one of the three basic browser dialog functions: alert (show), prompt (ask), confirm (yes/no).
"Welcome…"
The argument — what goes inside the popup
Any String (text in quotes) or variable you pass becomes the popup message.
Try: alert(42) — shows a number. alert(true) — shows "true". alert("Hello " + "World") — concatenates. The message box displays whatever you pass in, converted to a String.

understanding string concatenation — the + operator
"Welcome " + name
// name contains "Rahul"
// Result: "Welcome " + "Rahul" = "Welcome Rahul"
The + operator with strings joins them together. "Hello " + "World" → "Hello World". This is called concatenation.
Note the space after "Welcome " inside the quotes. Without it, the result would be "WelcomeRahul" — no gap between the word and the name.

What Is the DOM?
Document Object Model — the bridge between JavaScript and your HTML page
DOM stands for
Document  Object  Model
JavaScript controls HTML through the DOM
dom = family tree of your html
📄 document
└─ <html>
├─ <head>
│ └─ <title> SkillHub LMS
└─ <body>
├─ <h2 id="hero-title">
│ └─ Become a Full Stack…
└─ <button id="joinBtn">
└─ Join Now
the dom in plain english
HTML is a static document
When the browser reads your HTML, it builds a tree of objects in memory — one object per element.
JavaScript can modify any object in that tree
Change text, change styles, add elements, remove elements — all in real time without reloading the page.
The browser instantly shows the change
Users see the updated page immediately — no refresh needed.
Key mental model: Every HTML element on your page has a matching JavaScript object. If you can find that object (getElementById), you can change anything about it. This is what the next three slides are about.

anatomy — click each part
document
The entire HTML page as a JS object
document represents your whole HTML file. It's the root of the DOM tree — your starting point to find any element.
document is a global object — always available in the browser. You never create it; the browser creates it automatically when it loads your page.
.getElementById
Method — finds element by its id attribute
Searches the entire page for the element whose id matches what you pass in. Returns that element as a JS object.
If no element with that id exists, it returns null. IDs must be unique per page — two elements cannot share the same id. Other selectors: querySelector, getElementsByClassName.
("hero-title")
The id to search for — must match exactly
Must exactly match the id attribute in HTML — including capitalisation. "hero-title" ≠ "Hero-Title".
Always a String (in quotes). The # symbol used in CSS is NOT used here — getElementById("hero-title") not getElementById("#hero-title").
let heading =
Store the result in a variable
The found element is saved in heading. Now heading IS the <h2> element — you can read and modify it.
Think of heading as a remote control for the <h2>. heading.innerHTML changes its content. heading.style.color changes its colour. heading.style.display = "none" hides it entirely.
Always add an id to any HTML element you want to control with JavaScript. No id = JS cannot find it.

heading.innerHTML
innerHTML — the HTML content inside an element
Reading heading.innerHTML gives you the current text. Setting it (=) replaces everything inside the element.
innerHTML can include HTML tags too: heading.innerHTML = "<strong>Welcome</strong> to SkillHub" — the browser renders the bold tag. Use textContent if you only want plain text (safer and faster for text-only changes).
= "Welcome…"
Assignment — replace the content
The = sign sets innerHTML to the new string. The old content ("Become a Full Stack Developer") is replaced instantly.
This is why we say JS "mutates the DOM" — it modifies the live document in memory. The HTML file on disk is unchanged; only the browser's in-memory copy changes. Refresh reloads from disk, resetting the change.
Students immediately see: JavaScript changed HTML. This is the "aha" moment for DOM. Let every student try running it themselves before moving on.

connect in index.html
frontend/index.html — bottom of body
…<!-- Bootstrap JS -->
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.7/…"></script>
<!-- Our JavaScript -->
<script src="js/script.js"></script>
</body>
</html>
why before </body>?
The browser reads HTML top to bottom. JavaScript often needs to find HTML elements (buttons, headings, divs). If JS runs before the HTML loads, those elements don't exist yet and JS throws an error.

Placing <script> at the bottom of <body> ensures HTML loads first — then JS runs safely.

what is the console?
The Console is the developer's notebook. It shows messages that developers write — users never see it, only developers who press F12.

It's the most important tool for testing and debugging JavaScript. Whenever something isn't working, console.log() is how you find out why.
how to open the console
Press F12 (Windows) or Cmd+Option+J (Mac)
Right-click anywhere → Inspect → Console tab
Save script.js → refresh the browser → see your log
Pro tip: console.log() is the first thing every developer reaches for. Even senior engineers use it daily. Get comfortable with it — you'll use it throughout the entire MERN course.

Data Types
Data Types — String · Number · Boolean
Every value in JavaScript has a type — JavaScript uses the type to know what it can do
String
Text values — always in quotes. Can be words, sentences, names, emails.
let student = "Rahul";
Use for: names, courses, emails, messages
Number
Numeric values — no quotes. Can be whole numbers or decimals. Used for maths.
let age = 20;
Use for: age, price, count, duration
Boolean
Only two values — true or false. Used for yes/no questions and conditions.
let enrolled = true;
Use for: login status, enrolled yes/no

Explain
Line-by-Line Explainer
Click any row to expand — every JavaScript concept from today
console.log("…");
console.log()
Prints a message to the browser console (F12). Invisible to users.
console = built-in JavaScript object · log = a method on it · ("…") = what to print. You can log strings, numbers, variables, objects. Used for testing and debugging. Senior developers use it every day. You'll use it in every JS file you ever write.
let
let — declare a variable
Creates a new variable that can be changed later
In modern JavaScript (ES6+), use let for variables that may change. Use const for values that never change (we'll use const a lot in React). Avoid the old keyword var — it has confusing scoping rules. Rule: use const by default, switch to let only if you need to reassign.
trainer
Variable name
The label for the container — choose meaningful names
Good names: studentName, courseFees, isEnrolled. JS uses camelCase (first word lowercase, next words capitalised). Bad names: x, data, stuff. Variable names cannot start with a number, cannot contain spaces, and are case-sensitive (trainer ≠ Trainer).
= "Manish Rai"
= (assignment operator)
Puts the value on the right into the variable on the left
The single = in JS means "assign this value." It is NOT the same as equals in maths. In maths, 5 = 5 checks if they're equal. In JS, let x = 5 puts 5 into x. To check equality in JS, you use === (triple equals) which we'll learn in Hour 2.
"Manish Rai"
String value — text in quotes
Any text surrounded by " " or ' ' is a String
Both "double quotes" and 'single quotes' create strings. A third option: template literals with backticks ` ` (we'll use these in Hour 2 for combining variables with text). Always close what you open — an unclosed quote is a very common syntax error.
45 · true
Number + Boolean values
Numbers: no quotes. Booleans: exactly true or false (no quotes, lowercase)
JS Number can be integer (45) or decimal (12.5). Boolean is always exactly true or false — lowercase, no quotes. Writing "true" (with quotes) is a String, not a Boolean. This matters when you use it in if-statements in Hour 2.
;
Semicolon — ends a statement
Tells JavaScript "this instruction is complete"
Technically optional in modern JS (ASI — Automatic Semicolon Insertion adds them for you), but always write them. Missing semicolons are a common source of confusing bugs. In React and Node.js code you'll see semicolons everywhere — good habit to build now.

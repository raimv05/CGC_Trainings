anatomy label
.addEventListener
The Listener Method
Built-in JS method on every DOM element. "Listen for this event, then call this function."
Every HTML element in the DOM has addEventListener. You can listen on a button, a div, a paragraph — even document or window.
"click"
Event Type (string)
The name of the event to listen for. Must be a string in quotes. "click", "mouseover", "keydown", "submit" are common types.
Event names are case-sensitive and always lowercase. "Click" or "CLICK" won't work — it must be "click".
function() {…}
The Callback Function
The function that runs when the event fires. It only runs when the event happens — not immediately when addEventListener is called.
This is called a "callback" because you're telling addEventListener: "call this function back when the event happens." The browser holds on to it until it's needed.
comparison — old way vs modern way
❌ Old HTML way (avoid this)
<button onclick="alert('hi')">
Mixes HTML and JS — hard to maintain. Only supports one handler.
✅ Modern JS way (use this)
btn.addEventListener("click", fn);
Keeps JS separate. Can add multiple listeners to one element.
Always use addEventListener in real projects. The inline onclick= approach is outdated and makes code harder to manage as your project grows.

document.body
document.body — The Whole Page
document.body gives you the <body> element directly — no getElementById needed. Everything visible is inside body.
document.body is a shortcut. You can also do getElementById on the body if it has an id, but document.body is simpler and universally available.
.style.background
Inline Style via JS
Sets the CSS background property directly on the element as an inline style. This overrides any existing stylesheet rules.
CSS property names with hyphens become camelCase in JS: background-color → backgroundColor. Both background and backgroundColor work for setting a solid colour.

Feature 3 — Click to Increment
Available Courses: 6
total++
Increment Operator
total++ is shorthand for total = total + 1. Adds 1 to the variable every time it runs.
Other shortcuts: total-- (subtract 1), total += 5 (add 5), total *= 2 (multiply by 2). The ++ is called the "post-increment" operator.
.innerHTML = total
Update the DOM
Sets the content of the #count span to the current value of total. JS automatically converts the number to a string for display.
This is the complete pattern for live-updating numbers: keep the value in a variable, update the variable, then push the new value to the DOM with innerHTML.
Key insight: The variable total lives in JS memory. The #count span lives in the DOM. They're separate — you must manually sync them with .innerHTML = total every time the variable changes.

new Date()
Creating a Date Object
new Date() creates a Date object set to the current date and time. The "new" keyword creates an instance of the built-in Date class.
new Date() with no arguments = right now. new Date("2024-01-15") = a specific date. new Date(year, month, day) = also a specific date (month is 0-indexed).
.toDateString()
Human-Readable Format
Converts the Date object to a readable string like "Mon Jan 15 2024". Always English, always the same format across browsers.
For locale-aware formatting, use toLocaleDateString() — it shows the date in the user's regional format (e.g. DD/MM/YYYY in India vs MM/DD/YYYY in the US).
Notice this code has no addEventListener — it runs immediately when the <script> tag executes. The date is always current because new Date() captures the moment it runs.

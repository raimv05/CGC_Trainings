Part 1
Why Node.js?
Our LMS currently runs only in the browser
Ask the class:
Can JavaScript create files? Not in the browser.
Can it access the operating system? Not in the browser.
Can it build APIs? Not in the browser.
Node.js solves this. It takes the same JavaScript language and lets it run outside the browser — on your machine, with access to files, networks, and the operating system.

Line-by-Line — Node Fundamentals
Click any line to see what it actually does
node app.js
node app.js
Tells the Node runtime to execute app.js from top to bottom.
Unlike browser JS, there's no page to load — Node just runs the file directly and prints anything console.log()'d to the terminal.
npm init -y
npm init -y
Generates a package.json instantly, accepting all the default answers.
Without -y, npm init asks a series of questions (name, version, entry point...) one at a time.
module.exports
module.exports
Whatever is assigned here becomes the thing another file receives when it calls require() on this file.
You can export a single function, an object of many functions, a class, or any value at all.
require("./utils")
require("./utils")
Loads utils.js and returns whatever it exported.
The "./" prefix means "look in this same folder". Without it, Node assumes you mean an installed npm package instead of a local file.
let students = [];
let students = [];
An empty array that will hold every student object added during this session.
Because it's in memory only, this array resets every time the program restarts — that's exactly the gap fs.writeFileSync() closes later today.
require("fs")
const fs = require("fs")
Loads Node's built-in File System module — no installation needed, it ships with Node itself.
"fs" is a core module, unlike "./utils" which is a local file. Core modules never need a "./" prefix.

1function deleteStudent(name){
2students = students.filter(s => s.name !== name);
3}
try it live — the same students array from the previous slide
0 students
filter() never mutates the original array — it builds and returns a brand new one containing everyone except the removed student. That's why students = students.filter(...) reassigns the variable.
2nd 90 min
Save to File
fs.writeFileSync() turns the array into a real JSON file
student.js
1fs.writeFileSync(
2"students.json",
3JSON.stringify(students,null,2)
4);
live — students.json (updates as you add/delete students on the previous slides)
students.json is empty — go add a student first
Why JSON.stringify? fs.writeFileSync only accepts text. Passing the array directly would write the useless string "[object Object]" — stringify() converts it into real, readable text first.
2nd 90 min
Read File Back
fs.readFileSync() + JSON.parse() bring the data back to life
student.js
1const data = JSON.parse(
2fs.readFileSync("students.json","utf8")
3);
4console.table(data);
fs.readFileSync()
Reads the file's contents back in as plain text.
JSON.parse()
Converts that text back into a real, usable array.
Restart the program and the students are still there. That's the exact same "save it → load it" pattern from Local Storage — just written with Node's file system instead of the browser's.

Part 3
Install, Configure & Initialize
Get Git ready, then turn the LMS folder into a repository
1 — verify install
terminal
$ git --version
git version 2.44.0
2 — configure identity
terminal
$ git config --global user.name "Your Name"
$ git config --global user.email "your@email.com"
This runs once per machine. Every commit you make afterward gets stamped with this name and email — it's how Git (and GitHub) know who did what.
3 — initialize the repository
~/skillhub-lms
$ git init
Initialized empty Git repository in ~/skillhub-lms/.git/
A hidden .git folder appears. That's the entire local repository — every commit, branch and history entry lives inside it. Delete it, and Git forgets everything.

Explain
Line-by-Line — Core Git Commands
Click any command to see what it actually does
$ git init
git init
Turns the current folder into a Git repository by creating a hidden .git folder.
This only runs once per project. Everything Git needs to track history lives inside .git — delete that folder and the project loses all version history.
$ git status
git status
Shows the current state of every file — untracked, modified, staged, or nothing to commit.
Run this constantly. It's the safest way to know exactly what Git will do before you commit anything.
$ git add .
git add .
Stages every changed file in the current folder, preparing them for the next commit.
The "." means "everything in this directory". You can also stage a single file with git add filename.js.
$ git commit -m "..."
git commit -m "message"
Saves the staged files as a permanent snapshot, labeled with the message you provide.
-m lets you write the message inline. Without it, Git opens a text editor and waits for you to type one there instead.
$ git remote add origin <url>
git remote add origin <url>
Links your local repository to a GitHub repository, naming that link "origin".
"origin" is just a nickname for the remote URL, so future commands can say origin instead of the full GitHub link every time.
$ git push -u origin main
git push -u origin main
Uploads local commits to GitHub, onto the main branch.
-u remembers this origin/main pairing, so afterward a plain git push is enough — no need to repeat the destination every time.

Merge — Bringing the Feature Back
Merging combines a branch's work into another branch

$ git merge feature-login
Updating a1b2c3d..e4f5g6h
Fast-forward
the feature development workflow
1
Branch off main
git branch feature-name, then switch to it.
2
Build the feature
Edit, add, commit — as many times as needed.
3
Switch back to main
git switch main.
4
Merge it in
git merge feature-name — main now has the feature too.
5
Push
git push — the updated main goes to GitHub.

In-Class Activity
Students create: feature-dashboard
1. Add one feature to the SkillHub LMS (anything small — a heading, a badge, a card)
2. Commit the change with a clear message
3. Switch back to main and merge feature-dashboard into it
4. Push the final, merged main branch to GitHub

terminal
$ git branch feature-dashboard
$ git switch feature-dashboard
// make your change, then...
$ git add .
$ git commit -m "Added dashboard section"
$ git switch main
$ git merge feature-dashboard
$ git push

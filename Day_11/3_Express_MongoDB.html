Day 14 · Part 1
Why Express?
3 Hours · Learning Outcomes: Server, Routing, Middleware, Request, Response, JSON, HTTP Status Codes
Project Goal
Create the first backend server for SkillHub LMS.
Start with a question
Yesterday we wrote plain Node.js. Can Node create APIs easily? Yes — but it takes too much manual code for routing, parsing request bodies, and handling every URL by hand.
Express is a minimal web framework built on top of Node.js. It gives us routing, middleware, and clean request/response handling out of the box — so we stop reinventing the wheel for every route.

  • POST /courses

Day 14
Creating Your First Express Server
app.js — three lines to get a server listening
app.js
const express = require("express");

const app = express();

const PORT = 5000;

app.listen(PORT, () => {
  console.log(`Server Running on Port ${PORT}`);
});
Terminal
$ node app.js
Server Running on Port 5000
localhost:5000
Nothing happens...
Cannot GET / — we haven't defined a route yet.
Starting the server ≠ handling requests. Express needs a route to know what to respond with. That's next.

Day 14 · Part 2
What is Middleware?
Airport security — every passenger passes through checks before boarding
Passenger ↓ Security Check ↓ Flight
Request ↓ Middleware ↓ Route
1
Request Arrives
Browser sends GET /courses to the server.
2
app.use(express.json())
Built-in middleware — parses incoming JSON so req.body isn't undefined.
3
Logger Middleware
console.log(req.method, req.url) then calls next() to continue.
4
Route Handler
app.get("/courses", ...) finally runs and sends the response.
app.js
app.use(express.json());

app.use((req,res,next)=>{
  console.log(req.method);
  console.log(req.url);
  next(); // continue to the next step
});
Without express.json(), a POST request's body cannot be read — req.body would be undefined.

Day 15 · Part 2
PUT & DELETE — Completing the CRUD
This is the same `courses` array from Day 14 — now update and delete are live too
app.js
app.put("/courses/:id", (req,res)=>{
  courses[req.params.id] = req.body;
  res.json({ message: "Updated" });
});

app.delete("/courses/:id", (req,res)=>{
  courses.splice(req.params.id, 1);
  res.json({ message: "Deleted" });
});
Full CRUD Tester — Postman-style
Use PUT to edit a row inline, DELETE to remove it.

// Create
await Course.create(req.body);

// Read
await Course.find();

Every Mongoose model method returns a Promise — always await it inside an async route handler.
Day 17
Folder Structure — Separation of Concerns
Why we split a backend into layers
backend/
controllers/
models/
routes/
config/
middleware/
app.js
Why separate concerns?
  • models/ — Mongoose schemas only
  • controllers/ — business logic (what a route does)
  • routes/ — which URL maps to which controller
  • middleware/ — auth, logging, validation
  • config/ — DB connection, env setup

<a href="#">
Anchor Tag — Hyperlink
Creates a clickable link. href="#" links to the top of the same page (placeholder).
Later we'll replace # with a real URL — href="courses.html" to link to the courses page we built in Day 2 Hour 2. For now, # is a safe placeholder that makes the link clickable without navigating away.
nav a
Descendant Selector
Targets all <a> tags that are inside a <nav> element.
This is a descendant selector — the space between nav and a means "any <a> that is a child of <nav>." It will NOT affect <a> tags elsewhere on the page (e.g. in the footer). Very useful for targeting specific groups of links.
Why href="#"? The hash symbol links to the current page's top. It makes the element behave like a real link (clickable, cursor changes, keyboard focusable) while we're still building. We'll add real page paths later in the MERN project.

the 3 properties explained
background
Sets the background colour of the nav bar. #1e293b is the same dark colour as the header — creating a consistent dark top section. We could also write background-color — both work for solid colours.
padding
Adds inner space around the nav content. 15px applies to all four sides. Without it the links are squished against the edges. Padding = space inside the element.
text-align
Controls horizontal alignment of inline content (text, links) inside the element. center places all nav links in the middle. Other values: left, right, justify.

color: white
color — Text Colour
Changes the text colour of the links to white
By default, browser links are blue. Setting color: white overrides this. You can use colour names (white, red, blue), hex codes (#ffffff), or rgb values (rgb(255,255,255)). All three mean the same colour: white.
text-decoration: none
text-decoration — Remove Underline
Removes the default blue underline that browsers add to all links
Browsers automatically add text-decoration: underline to all <a> tags. Setting it to none removes this. Other values: underline, overline, line-through (for strikethrough text). Most professional navbars remove the underline and handle hover separately.
margin: 15px
margin — Outer Space
Adds 15px of space OUTSIDE each link — creating gaps between them
Margin = outer space (space around the element, pushing other elements away). margin: 15px applies 15px to all four sides. Compare with padding (inner space): padding pushes content away from the element's own edges; margin pushes the element away from other elements.

Hover Effect — CSS Reacts to the User
The :hover pseudo-class — CSS that triggers on mouse-over
❓ Can CSS react when a user does something?
Yes! CSS has pseudo-classes — special selectors that activate based on user interaction. :hover activates when the mouse is on top of an element. No JavaScript needed — this is pure CSS!

Explain
Property Deep-Dive
Click each row to understand text-decoration, margin, and color in depth
text-decoration
text-decoration — Line Decorations on Text
Controls underlines, overlines, and strikethroughs on text
Values: none (remove underline from links), underline (add underline — default for links), line-through (strikethrough, like a deleted price), overline (line above text). In navbars we always set text-decoration: none to remove the browser's default link underline and get a clean look.
margin
margin — Outer Space Around an Element
Space outside the element's border — pushes neighbours away
1 value: margin: 15px — all 4 sides equal. 2 values: margin: 10px 20px — top/bottom 10px, left/right 20px. 4 values: margin: top right bottom left (clockwise). Remember: Margin = outside, Padding = inside. On inline elements like <a>, margin adds horizontal breathing room between links.
color
color — Text Colour
Sets the colour of the text (not the background)
Three ways to write colours in CSS: color: white (named colour — 140+ names available), color: #facc15 (hex code — 6 characters after #, pairs of RR GG BB), color: rgb(250, 204, 21) (red, green, blue values 0–255). All three are equivalent. Hex codes are most common in professional code.
:hover
:hover — Pseudo-Class
Activates CSS rules when the mouse is over the element
A pseudo-class is a keyword added to a selector that specifies a special state. :hover fires when the user's mouse cursor is positioned over the element. It only changes the element's appearance — it does NOT move or resize it. Other pseudo-classes: :focus (keyboard focus / input active), :active (element is being clicked), :visited (link was already visited).

why use a class instead of just section { }?
❌ If we wrote section { }
It would apply to every <section> on the page — the courses section, testimonials section, about section. We don't want that!
✅ With class="hero" and .hero { }
Only the section with class="hero" gets these styles. Other sections remain unaffected. Precision targeting.
📌 Rule
HTML: class="hero" → CSS: .hero { } (dot prefix). The dot in CSS means "target elements with this class name."

.hero
Class Selector
The dot (.) targets elements with class="hero" — only that section
A class selector starts with a dot. .hero means "apply these styles to any element that has class="hero" in the HTML." We added that class to the <section> in the previous slide. Classes can be reused — if we add class="hero" to another element, it gets the same styles too.
.hero h2
Descendant Class Selector
Targets <h2> elements that are INSIDE the .hero section
This is a descendant selector using a class. It reads: "target any <h2> that is a descendant of an element with class hero." This means other h2 tags on the page (in other sections) are NOT affected. Very precise targeting — best practice.
font-size: 40px
font-size — Control Text Size
Sets how big the text is. 40px for the hero heading makes it dominant and impactful.
px = pixels. The default browser h2 size is around 24px. Setting 40px makes it much larger and creates a strong visual hierarchy. Other units: rem (relative to root font size — scales with user preferences), em (relative to parent font size), % (percentage of parent). For now, px is simplest to understand.
margin-top: 20px
margin-top — Space Above an Element
Adds 20px of space specifically above the paragraph
You can target individual sides of margin: margin-top, margin-right, margin-bottom, margin-left. This gives finer control than margin which sets all four sides at once. Here, 20px top margin pushes the paragraph down from the heading, creating readable breathing room.

border: none
border: none — Remove Default Button Border
Browsers add a grey border to <button> elements by default — this removes it
Without border: none, the button would show the browser's default grey border around our blue background — looks messy. Other border values: border: 1px solid #000 (width style colour — the three-part shorthand). We'll use borders later when we want outlined/ghost buttons.
padding: 12px 25px
Two-Value Padding Shorthand
12px top/bottom · 25px left/right — makes the button tall enough to click
Two-value shorthand: first value = top and bottom, second value = left and right. So padding: 12px 25px = 12px padding top and bottom (making the button tall), 25px left and right (making it wide). This is the most common button padding pattern — taller than it is wide.
cursor: pointer
cursor: pointer — Show Hand Cursor
Changes the mouse cursor to a pointing hand when hovering over the button
By default <button> elements show a normal arrow cursor in some browsers. cursor: pointer forces the hand cursor — the universal visual signal that something is clickable. Always add this to styled buttons and links. Other cursor values: default, move, not-allowed, crosshair.
.btn:hover #1d4ed8
Hover State — Slightly Darker Blue
Hover darkens the button to give visual click feedback
#2563eb (normal) vs #1d4ed8 (hover) — the hover colour is just a shade darker. This is the standard UX pattern: hover = slightly darker/lighter than the default. It gives the user visual feedback that the button is interactive. No JavaScript — pure CSS.

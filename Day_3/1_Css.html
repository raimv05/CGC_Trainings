Question
Why Doesn't It Look Like Netflix?
Show students their Day 2 page — then ask this question
❓ Ask the class: "Your page has a header, nav, sections, footer — so does Netflix. Why does yours look different?"
HTML
Gives your page structure — like the skeleton of a building. Headings, paragraphs, lists, sections. No colours, no spacing, no fonts.
CSS
Gives your page beauty — like the paint, furniture, and decoration. Colours, fonts, spacing, layout, animations.
Netflix is also HTML + CSS. Their CSS file is just very, very detailed. 🎨
HTML = Skeleton
Bones & structure. Without it the page has no content at all.
CSS = Skin & Clothes
Colours, fonts, spacing, layout. Makes it look professional.
JS = Muscles
Interactivity & logic. We'll add this in Week 2 with React.
Key insight: You can apply completely different CSS files to the same HTML and get wildly different results. The structure stays — only the appearance changes. This is the power of separation of concerns.
Connect
Link CSS to HTML
Two steps: create the file, then connect it — do both now
Step 1 — Create the CSS file in VS Code
1 Right-click the frontend/ folder in VS Code → New Folder → name it css
2 Right-click the css/ folder → New File → name it style.css
3 The file is empty for now — we'll fill it in the next steps
step 2 — add <link> inside index.html <head>
frontend/index.html — <head> section
1<head>
2<title>SkillHub LMS</title>
3<link rel="stylesheet"
4href="css/style.css">
5</head>
Path: css/style.css means "look inside the css folder, find style.css." The path is relative to index.html.
step 3 — write first css rule to test
frontend/css/style.css
1body{
2background-color:#f4f4f4;
3}
Now save both files and refresh the browser.
The page background turns from white to a very light grey — CSS is working! 🎉
rel="stylesheet"
rel attribute
Tells browser: "this link is a stylesheet" — not an icon or other resource
Without rel="stylesheet" the browser loads the file but doesn't apply it as CSS. Always required with <link>.
href="css/style.css"
href attribute
The path to the CSS file — relative to index.html
css/style.css = look in the css folder, open style.css. If you rename or move the file without updating href, styles will stop loading.
Syntax
CSS Syntax — Anatomy of a Rule
Every CSS rule follows the same pattern — learn it once, use it forever
CSS Rule Anatomy — click labels for details
h1 {
color : blue ;
}
Selector
Which HTML element to style.

Examples: body h1 header p .class #id

Targets every matching element on the page.
Property
What to change about the element.

Examples: color background font-size padding margin

Always written in lowercase with hyphens.
Value
What to set the property to.

Examples: blue #2563eb Arial 20px center

Always followed by a semicolon ;
real example — h1 styling
style.css
1h1 {
2color: blue;
3}

Common mistakes: Missing semicolon ; after a value (styles below it may break) · Missing closing brace } · Typo in property name (e.g. colour instead of color). If CSS isn't working, check these first.
Style 1
Style the Body
background and font-family — affects the entire page
Add this to style.css — update what you already wrote
frontend/css/style.css
1body {
2background: #f4f6f9;
3font-family: Arial, sans-serif;
4}
background
Sets the background colour of the element. #f4f6f9 is a very light blue-grey — professional and easy on the eyes. Try these alternatives:
#f4f6f9 #ffffff #141414 #f0f9ff
font-family
Sets the font for all text. Arial, sans-serif means: "use Arial; if not available, use any sans-serif font." The fallback ensures text is always readable.

Style 2
Style the Header
background · color · padding · text-align — four new properties
Add this below the body rule in style.css
frontend/css/style.css — add below body { }
6header {
7background: #2563eb;
8color: white;
9text-align: center;
10padding: 20px;
11}
background: #2563eb
Sets a vivid blue background. #2563eb is Tailwind's blue-600 — widely used in modern UIs. Change this to try other colours: #dc2626 (red) · #059669 (green) · #7c3aed (purple)
color: white
Sets the text colour to white. Without this, dark text on a blue background is still readable — but white looks professional. Named colours like white, black, red work. So does #ffffff.
text-align: center
Centers all text horizontally. Values: left (default) · center · right · justify.
padding: 20px
Adds inner spacing — 20px of space between the edge of the header and its content. Without padding, the text sits flush against the edge. Always makes things look more polished.

Explain
Property-by-Property Explanation
Click any row to expand — cover all 6 properties from today
background: #f4f6f9
background — body
Sets the page background to a soft blue-grey
Can be a hex colour (#f4f6f9), a named colour (lightgrey), or even an image (url('bg.jpg')). The background shorthand covers both background-color and more.
font-family: Arial
font-family — body
Changes all text to the Arial typeface
Always provide a fallback: Arial, sans-serif. If Arial isn't on a device, the browser picks any available sans-serif font. System fonts (Arial, Georgia, Courier) are safe — they exist on every device.
background: #2563eb
background — header
Fills the header with a vivid blue colour
Hex colour codes: # followed by 6 characters (0–9, A–F). First two = Red, middle two = Green, last two = Blue. #2563eb = low red, medium green, high blue = blue. Try #dc2626 for Netflix red.
color: white
color — header
Sets the text colour inside the header to white
colour vs background: color = text colour. background = background fill. These are the two most commonly confused properties. Remember: text = color.
text-align: center
text-align — header
Centers the "SkillHub LMS" text horizontally
Values: left (default — text hugs the left edge) · center (centred in the container) · right (hugs the right edge) · justify (spreads to fill the full width like newspaper columns).
padding: 20px
padding — header
Adds 20px of inner space on all four sides
Padding = inside the element (between content and border). Margin = outside the element (space around it). Without padding, the "SkillHub LMS" text sits flush against the edge of the blue header — looks cramped. 20px gives it room to breathe.

Reference
CSS Properties Reference
All properties covered today — keep this open while coding
PropertyWhat it controlsExample valueUsed on
background Background colour (or image) of an element #f4f6f9 · white · red body, header, section, any
color Text colour white · #1a1a1a · blue header, p, h1, any
font-family Typeface / font used for text Arial, sans-serif · Georgia, serif body (cascades down)
padding Inner spacing (between content and border) 20px · 10px 20px header, section, nav
text-align Horizontal alignment of text center · left · right header, section, p
font-size Size of text 16px · 2rem · 1.5em h1, h2, p, any
margin Outer spacing (space around the element) 10px · 0 auto · 20px 0 section, h2, p
Padding vs Margin: Padding = space inside (between content and edge). Margin = space outside (between this element and others). Remember: Padding is a Pillow (inside), Margin is Moat (outside).

Section with a Class
A semantic container for a page area. The class="about" lets CSS target it specifically.
Without a class, we'd have to write section { } which would affect ALL sections. The class gives us a name to hook into. This is the same pattern as .hero from Hour 2.

margin vs padding — the key difference
margin: 30px
Outer space — pushes the whole section away from surrounding elements. The 30px gap between the page edge and the white card comes from margin.
padding: 30px
Inner space — pushes the text away from the card's own edge. Without padding, the "About Us" heading would touch the white card border.
Memory trick: Padding is the pillow inside a sofa cushion — it's internal comfort. Margin is the gap between cushions — it's space between separate things.

display: flex
display: flex
Turns the container into a Flexbox — its direct children automatically line up in a horizontal row.
By default, HTML elements stack vertically (block layout). display:flex switches the container to Flexbox mode. Its direct children (the .card divs) then lay out in a row from left to right. This is the single most powerful layout switch in CSS.
gap: 20px
gap
Sets the space between flex items. Much cleaner than adding margin to each card individually.
gap only creates space between items — not before the first one or after the last one. Without gap, the three cards would be jammed together. You can also write gap: 20px 10px to set row-gap and column-gap separately.
justify-content: center
justify-content
Controls how flex items are positioned along the main axis (horizontal row). center pushes all cards to the middle.
Other common values: flex-start (left-aligned), flex-end (right-aligned), space-between (first at start, last at end, equal gaps between), space-around (equal gaps around each item). We use center here to keep the course cards in the middle of the page.

border-radius
Rounds the corners of the card. 10px = gentle curve. Try 50% to make a circle. 0 = sharp square corners.
box-shadow
4 values: 0 x-offset · 2px y-offset · 8px blur · rgba(0,0,0,0.2) colour. Creates the floating card look.
width: 250px
Fixes each card at 250px wide. Without this, cards would stretch to fill the container. Fixed width keeps all cards the same size.
box-shadow breakdown: box-shadow: 0 2px 8px rgba(0,0,0,0.2) — 0 = no left/right offset · 2px = 2px below · 8px = how soft/blurry · rgba(0,0,0,0.2) = black at 20% opacity. Increase the last number (0.4, 0.6) for a darker shadow.

.card:hover
:hover Pseudo-class
CSS that only applies when the mouse pointer is over the element. Same concept as nav a:hover from Hour 2.
The colon : signals a pseudo-class — a state that the element can be in. Other pseudo-classes: :focus (element is focused), :active (element is being clicked), :first-child (first among siblings). All of these can trigger separate CSS rules.
transform: translateY(-5px)
transform: translateY()
Moves the element along the Y axis (vertical). Negative = upward. -5px lifts the card 5 pixels up.
transform moves the element visually without affecting layout — surrounding elements don't shift. Other transform functions: translateX() (horizontal), rotate() (spin), scale() (resize). Combine them: transform: translateY(-5px) scale(1.02).
Explain
Property Deep-Dive
box-shadow · border-radius · transform — click each row to expand
box-shadow
box-shadow
Adds a drop shadow behind an element — creating a sense of depth and lifting the card off the page.
box-shadow: X Y Blur Spread Color. Our value 0 2px 8px rgba(0,0,0,0.2) means: 0 left/right offset, 2px downward, 8px blur radius, semi-transparent black. You can layer multiple shadows: box-shadow: 0 2px 8px rgba(0,0,0,.1), 0 4px 20px rgba(0,0,0,.05). Inner shadow: inset 0 2px 4px rgba(0,0,0,.1).
border-radius
border-radius
Rounds the corners of an element. Works on any element that has visible borders or a background.
Values: a single value applies to all 4 corners equally. You can target individual corners: border-radius: 10px 0 0 10px (top-left, top-right, bottom-right, bottom-left). border-radius: 50% makes a perfect circle if width = height. Widely used for buttons, cards, avatars, and pill-shaped badges.
transform
transform
Visually moves, rotates, scales, or skews an element without disrupting the document flow.
Because transform is visual-only, the browser doesn't re-flow the page — making it extremely fast to animate. Works with CSS transitions and animations: transition: transform 0.2s ease turns any transform into a smooth animation. Common functions: translate(x, y), rotate(deg), scale(n), skew(deg). Our card uses translateY(-5px) on hover for the lift.
display: flex
display: flex (recap)
Turns an element into a flex container — children line up in a row and can be aligned, distributed, and spaced with Flexbox properties.
Flexbox is a full layout system. Key properties on the container: flex-direction (row/column), justify-content (main axis), align-items (cross axis), flex-wrap (allow wrapping). Key property on items: flex: 1 (fill available space equally). We'll use more of these in React and the full MERN project.

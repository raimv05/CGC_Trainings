purpose of each item
index.html — The homepage of SkillHub. Every website starts with an index.html. The browser opens this file first.
assets/ — Where we'll store images, CSS files, and JavaScript files used by the frontend.
Why index.html? Web servers look for a file named index.html by default. When you visit www.skillhub.com, the server automatically sends index.html to the browser.

Learn
Line-by-Line Explanation
Click any row to expand the explanation
<!DOCTYPE html>
DOCTYPE Declaration
Tells the browser this is an HTML5 document
Without this, older browsers may go into "quirks mode" and render your page incorrectly. Always the very first line.
<html lang="en">
Root Element
The container for the entire webpage
lang="en" tells browsers and screen readers the language is English. Everything in your page lives inside this tag.
<head>
Head Section
Contains metadata — not visible on the page
Holds the title, charset, viewport settings, and links to CSS files. Think of it as the "brain" of the page — invisible but important.
<meta charset="UTF-8">
Character Encoding
Supports all languages including Hindi, Arabic, emoji
UTF-8 covers over 1 million characters. Without this, special characters may show as ??? or garbled symbols.
<meta name="viewport">
Viewport Meta Tag
Makes the page responsive on mobile devices
width=device-width means "use the screen's actual width." initial-scale=1 means "no zoom by default." Essential for mobile-friendly pages.
<title>SkillHub</title>
Page Title
Text shown in the browser tab
This is also what Google shows as the link title in search results. Type something meaningful like your project name.
<body>
Body Section
Everything visible on the webpage lives here
Headings, paragraphs, images, buttons, lists — all visible content goes inside <body>. This is what users actually see.
<h1>Welcome…</h1>
Heading 1
The biggest, most important heading on the page
There should be only one <h1> per page. Search engines treat it as the main topic. Shown in large bold text.
<p>Learn…</p>
Paragraph
A block of text with spacing above and below
Use <p> for body text, descriptions, and any prose content. The browser adds automatic top and bottom margin.

Tags
HTML Tags Reference
Every tag used today — keep this open while you code
<h1>–<h6>
Headings
H1 is the biggest (page title). H2 for sections. H3 for sub-sections. H4–H6 go smaller. H1 should appear only once per page.
<p>
Paragraph
A block of text. Browsers automatically add spacing before and after each paragraph. Use for all body text.
<ul>
Unordered List
Creates a bullet-point list. Contains <li> items. Use when the order of items doesn't matter (e.g. skills, features).
<ol>
Ordered List
Creates a numbered list (1, 2, 3…). Contains <li> items. Use when order matters (e.g. steps, rankings).
<li>
List Item
Always goes inside <ul> or <ol>. Each item gets its own <li> tag. Example: <li>MERN Stack</li>
<img>
Image
Displays an image. Self-closing — no </img> needed. Example: <img src="photo.jpg" alt="My photo"> · Always include an alt attribute for accessibility.
<a>
Anchor / Link
Creates a clickable hyperlink. Example: <a href="https://skillhub.com">Visit Us</a> · href sets the destination URL.

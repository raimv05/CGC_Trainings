Part 2 · 20 min
Arrow Functions
The most-used ES6 feature — replaces the function keyword in most cases
❌ Old way
students.forEach(function(student) {
console.log(student);
});
✅ New way
students.forEach(student => {
console.log(student);
});
what changed — spot the differences
1
The function keyword disappearsfunction(student) becomes just (student) — or student without parentheses if there's only one parameter.
2
An arrow => replaces itThe "fat arrow" sits between the parameters and the function body — that's where arrow functions get their name.
3
Behaviour is identicalBoth versions loop through the students array and log each one — arrow functions don't change WHAT the code does, only HOW it's written.
You've already used callback functions everywhere — addEventListener, forEach, the submit handler. Arrow functions are simply a shorter way to write those same callbacks.
Part 2 · continued
Arrow Function Syntax Rules
Click each form to see how it differs — there are 4 valid shapes depending on parameters and body
() => {...}
No Parameters
Empty parentheses when the function takes no arguments. Example: () => { console.log("hi"); }
You always need () even with zero parameters — it's the only form where parentheses are mandatory regardless of parameter count.
student => {...}
One Parameter — Parentheses Optional
With exactly ONE parameter, you can drop the parentheses entirely. student => {...} is the same as (student) => {...}
Both are valid — many style guides prefer keeping the parentheses for consistency, but dropping them is common in real code, especially for single-line arrow functions.
(a, b) => {...}
Multiple Parameters — Parentheses Required
With TWO or more parameters, parentheses are mandatory. (a, b) => a + b
This is a common error source — students forget the () are required once you have more than one parameter, since one-parameter functions let you skip them.
name => `Hi ${name}`
Implicit Return — No Braces, No "return"
If the function body is a SINGLE expression, you can drop the curly braces {} AND the return keyword. The value after => is automatically returned.
This only works for a single expression. If you need multiple lines or statements, you must use { } and an explicit return keyword.
Common mistake: writing name => { `Hi ${name}` } with braces but no return — this returns undefined because braces require an explicit return.
Rule of thumb: one-liner = no braces, no return. Multiple lines = use braces AND return.
Part 3 · 20 min
Destructuring — Object Unpacking
Pull multiple properties out of an object in one clean line
❌ Current way
student.name
student.email
student.course
Repeats "student." three times — every property access is a separate lookup
✅ Modern way
const {name, email, course} = student;
One line — "unpacks" all 3 properties into standalone variables
visualising the unpack
student = {
name: "Rahul",
email: "rahul@x.com",
course: "MERN Stack"
}
⇒
name = "Rahul" email = "rahul@x.com" course = "MERN Stack"
Object unpacking. The variable names inside {} must match the property names exactly — destructuring matches by NAME, not by position (unlike array destructuring, which matches by order).
You'll use destructuring constantly once you reach React — it's the standard way to pull values out of props and state objects.
Part 4 · 15 min
Spread Operator — ...
Expands an array's items so they can be combined with others
the code
script.js
1 let frontend = ["HTML", "CSS"];
2 let backend = ["Node", "Express"];
3
4 let courses = [...frontend, ...backend];
...frontend
"Spread" the array open
The three dots "unpack" the array's individual items rather than nesting the whole array as one element.
Without ..., writing [frontend, backend] would give you a nested array: [["HTML","CSS"], ["Node","Express"]] — not what we want.
merging arrays — visualised
HTMLCSS
+
NodeExpress
↓ spread & merge ↓
HTMLCSSNodeExpress
courses = ["HTML","CSS","Node","Express"] — one flat array
Merge Arrays. Spread is also used to copy arrays/objects ([...original]) without mutating the original — important for things like React state updates later.
Part 5 · 20 min
Rest Operator — ...
Same three dots, opposite job — collects multiple arguments INTO one array
the code
script.js
1 function totalStudents(...students) {
2 console.log(students);
3 }
4
5 totalStudents("Rahul", "Priya", "Arjun");
...students
"Rest" of the arguments
Collects any number of arguments passed to the function into a single array called students — works with 1, 3, or 100 arguments.
Rest must be the LAST parameter in a function. You can have regular parameters before it: function intro(greeting, ...names) is valid.
collecting arguments — visualised
totalStudents("Rahul", "Priya", "Arjun")
"Rahul" "Priya" "Arjun"
↓ ...students collects them ↓
students = ["Rahul", "Priya", "Arjun"]
Collect multiple values. Rest is perfect when you don't know in advance how many arguments will be passed — no need for a fixed parameter list.
Spot the Diff
Spread vs Rest — Don't Confuse Them
Same ... syntax. The direction of data flow tells you which one it is.
SPREAD — unpacks (one → many)
[...frontend, ...backend]
Used when calling / building something — inside array literals, object literals, or function CALLS. Takes one array and spreads its contents out.
REST — collects (many → one)
function total(...students) {}
Used when defining something — inside a function's parameter list. Takes many arguments and collects them into one array.
QuestionSpreadRest
Where does it appear?Array/object literals, function callsFunction parameter list only
Direction of dataArray → individual itemsIndividual items → array
Position ruleCan appear anywhereMust be the LAST parameter
Memory trick"Spread it OUT""REST collects what's left"
Quick test for students: if the ... is inside [ ] or ( ) when CALLING a function — it's spread. If it's inside ( ) when DEFINING a function — it's rest.

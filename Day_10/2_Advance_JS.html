.classList.toggle
Toggle — add if absent, remove if present
If the "dark" class is NOT on body → adds it. If it IS on body → removes it. One method, two jobs — perfect for on/off features.
You already know classList.add() from form validation. .toggle() combines add + remove into one call, checking which action is needed automatically.
"dark"
The CSS Class We Toggle
When this class is added to <body>, all .dark rules in CSS activate. This makes the whole page switch themes instantly.
Using a class on body means every element on the page can respond to it — just write .dark .navbar, .dark .card, etc. in CSS.

how cascading dark mode works
1
JS adds "dark" to <body>The <body> element now has class="dark" in the DOM
2
CSS .dark rule activatesbackground:#222 and color:white apply to the whole body
3
Descendant selectors kick in.dark .card, .dark table, .dark input — all target elements INSIDE .dark body
4
JS toggle removes itClick again → "dark" class is removed → all light styles return instantly
Pro tip — save dark mode preference: After toggling, call localStorage.setItem("darkMode", "on") — then on page load, check and re-apply it. Users hate having to toggle dark mode every visit.
This pattern — classList.toggle + a CSS class on body — is how sites like GitHub, VS Code, and Twitter implement dark mode.

2

4 const toast = document.createElement("div");
5

7 toast.className =

11 toast.innerHTML = message;
12

setTimeout
Auto-dismiss after 3 seconds
setTimeout(fn, 3000) runs the function once after 3000ms (3 seconds). Here it calls toast.remove() to delete the element from the DOM.
Unlike setInterval (repeats forever), setTimeout runs exactly once. The 3000ms delay gives users enough time to read the message before it disappears.
type = "success"
Default Parameter (ES6)
If showToast("message") is called without a second argument, type defaults to "success". Another ES6 feature — right in the function signature.
Default parameters mean: showToast("Done!") → green toast. showToast("Failed!", "danger") → red toast. The caller decides.

JSON.stringify
Object → JSON String
Converts a JavaScript object/array into a JSON-formatted string. The null, 2 arguments add indentation — making it human-readable.
null = no replacer function. 2 = 2-space indent per level. Without these: {"students":[{"name":"Rahul"}]} — one dense line. With them: the formatted version shown here.
new Blob
Binary Large Object
A Blob wraps the JSON string into a file-like object the browser can treat as a downloadable file with a specific MIME type.
URL.createObjectURL(blob) generates a temporary URL like blob:http://... that points to that file in memory — the <a> tag then navigates to it, triggering a download.

📥 Import Data — Load JSON File
Students choose a JSON file → the dashboard reads and updates itself

4 class="form-control">
javascript — utils.js
js/utils.js
1 document.getElementById("importFile")
2 .addEventListener("change", function(e) {
3
4 // Get the selected file
5 const file = e.target.files[0];
6 const reader = new FileReader();
7
8 // When file is read, parse the JSON
9 reader.onload = function(ev) {
10 const data = JSON.parse(ev.target.result);
11 saveToStorage("students", data.students);
12 saveToStorage("courses", data.courses);
13 showToast("📥 Data imported!", "success");
14 renderAll(); // re-draw dashboards
15 };
16 reader.readAsText(file);
17 });
how the import flow works
1
User picks a .json fileThe <input type="file"> fires a "change" event with the selected file object.
2
FileReader reads itFileReader is a browser API — like Date, it uses the new keyword. .readAsText() reads the file contents as a string.
3
onload fires when readyReading is async — .onload runs when the file is fully read. ev.target.result contains the raw JSON string.
4
JSON.parse converts itTurns the raw string back into a JavaScript object. Then save to localStorage and re-render both dashboards.
Export + Import together give SkillHub a simple backup system — admins can back up data to a file and restore it later. This is a genuine, production-grade feature.

the dry principle — don't repeat yourself

// in student.js

localStorage.setItem("courses", JSON.stringify(list));

function saveToStorage(key, data) {

// student.js — use it
saveToStorage("students", list);
// course.js — use it
saveToStorage("courses", list);

Part 1 · 15 min
Introduction to the DOM
Yesterday we created static HTML. Today, HTML learns to move.
Can HTML create a brand new Course Card by itself, with no JavaScript at all?
No.
HTML is static — it describes what's already there. It has no way to react to a click or generate a new card on demand. JavaScript is what manipulates HTML after the page has loaded, by talking to something called the DOM.
HTML
The page's starting structure
DOM
HTML turned into a live object tree
JavaScript
Reads and edits that tree
Updated HTML
What the visitor actually sees
DOM = Document Object Model. The moment the browser loads your HTML, it builds a "live" tree version of it in memory. JavaScript never edits the HTML file itself — it edits this tree, and the browser instantly redraws the screen to match. That's how a new course card can appear without a page reload.
Part 2 · 15 min
Selecting Elements
Before you can change something on the page, JavaScript has to find it first
getElementById()
Finds ONE element by its unique id. Fastest, most common selector.
querySelector()
Finds the FIRST element matching any CSS selector — #id, .class, tag.
querySelectorAll()
Finds ALL matching elements, returned as a list you can loop through.
Reference example

Explain
Line-by-Line — DOM & Rendering
Click any row to expand — every piece of new syntax from the First 90 Minutes
.getElementById()
getElementById(id) — find one element
Returns the single element with that exact id, or null if none exists
This is how addCourse() reads what the visitor typed: document.getElementById("courseName").value grabs the current text inside that input box.
.addEventListener()
addEventListener(event, function) — react to actions
Runs a function automatically whenever a specific event happens on that element
"click" is the event, addCourse is the function to run. Notice we pass addCourse — not addCourse() — we want JavaScript to call it later, not run it immediately.
courses.push()
push(item) — add to the end of an array
Adds the new course object as the last item in the courses array
Exactly the same method you used with the Student Dashboard — arrays of objects always grow this way.
courses.forEach()
forEach((item, index) => {...}) — loop through every item
Runs the given function once per course, giving you both the course object and its position
The index parameter is what lets the Delete button on each card know exactly which course to remove later.
`...`
Template Literals — backtick strings
Strings written with backticks (`) instead of quotes — they can span multiple lines and embed variables
This is what makes it possible to write readable, multi-line HTML directly inside JavaScript instead of ugly string concatenation with +.
${ }
${expression} — interpolation
Inserts the value of a variable or expression directly into a template literal string
${course.name} is replaced with the actual course name at the moment the code runs — this is how one template builds a different card for every course.
.innerHTML
.innerHTML — replace what's inside an element
Sets the HTML content of an element, replacing everything that was there before
Setting courseContainer.innerHTML = html swaps in the entire freshly-built set of cards in one go, rather than adding them one at a time.

Delete Course
splice() removes exactly one item — the one whose index matches the button clicked
Write deleteCourse(index)
frontend/js/script.js
function deleteCourse(index) {
courses.splice(index, 1);
displayCourses();
saveCourses(); // coming up in Part 3
}
why splice(index, 1)
Remember: the button's onclick already carried the exact index from forEach(). splice(index, 1) means "starting at this position, remove exactly 1 item" — every course after it shifts down automatically.

document.getElementById("searchCourse")
.addEventListener("keyup", function() {
const keyword = this.value.toLowerCase();
const filtered = courses.filter(course =>
course.name.toLowerCase().includes(keyword)
);
displayCourses(filtered);
});
update displayCourses() to accept data
frontend/js/script.js
function displayCourses(data=courses) {
let html = "";
data.forEach((course,index) => { /* ...same as before */ });
document.getElementById("courseContainer").innerHTML = html;
}
data = courses is a default parameter. Call displayCourses() with nothing and it renders the FULL list, exactly as before. Call it with a filtered array and it renders only that — one function, two jobs, no duplicate code.
Reusable functions are what separate "code that works once" from "code that scales." This same displayCourses(data) will also render results after Local Storage loads, and after any future filter you add.

2nd 90 Min · Part 3 · 25 min
Local Storage
Students already know arrays. Now — what happens after a refresh?
Add three courses in the demo you just tried. Now press F5. What happens?
Everything disappears.
The courses array only exists in RAM while the page is open. A refresh restarts JavaScript from the top — let courses = [] runs again, empty. We need somewhere the data can survive a reload: Local Storage.
setItem(key, value)
Saves a string under a named key
getItem(key)
Reads it back — even after a reload
JSON.stringify / parse
Converts our array of course objects to/from text
2nd 90 Min · Part 3
Save & Load Courses
One function to save, one line to load — called at exactly the right moments
save — saveCourses()
frontend/js/script.js
function saveCourses() {
localStorage.setItem(
"courses",
JSON.stringify(courses)
);
}
Call saveCourses() in two places: right after courses.push() inside addCourse(), and right after courses.splice() inside deleteCourse(). Miss one, and that action won't survive a refresh.
load — at the very top of the file
top of frontend/js/script.js
let courses =
JSON.parse(
localStorage.getItem("courses")
) || [];
displayCourses();
|| [] protects the very first visit, when nothing has been saved yet.

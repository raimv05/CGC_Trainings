Part 3 · 20 min
The Object Problem
Can an array store a student's name, email, phone, AND course together?
Can a single Array hold a student's name, email, phone, and course — connected together?
trying with an array — confusing!
let student = ["Rahul","rahul@gmail.com","MERN"];
console.log(student[0]); // Is this the name? Or the email? Unclear!
No — not cleanly.
An array only has numeric positions (0, 1, 2…). You'd have to remember that index 0 = name, index 1 = email. Confusing and error-prone.
We need an Object — labelled data instead of numbered data.
Array — numbered, unclear
0Rahul
1email
2MERN
"Is index 0 the name or the ID? You have to remember."
Object — labelled, clear
{
name: "Rahul"
email: "…"
course: "MERN"
}
"name is obviously the name. Self-explanatory."

object syntax basics
{ } creates an object — curly braces (not square brackets like arrays).
Each line is a key: value pair, separated by commas — except the last one.
name, email, course are called keys or properties.
Console
▶{name: "Rahul", email: "rahul@gmail.com", course: "MERN"}
Array vs Object — when to use which: Array = a LIST of similar things (many students). Object = ONE thing with MANY properties (one student's details). We'll combine both shortly.

Explain
Line-by-Line — Arrays & Objects
Click any row to expand — every concept from Parts 1–3
let x = []
[] — empty array
Square brackets create an array. Empty means zero items so far.
An array is an ordered LIST. Items keep the order you added them in. You'll add to it using push() or by writing values directly: let courses = ["HTML", "CSS"].
.push(value)
push() — add to the end
Adds a new item at the end of the array, growing its length by 1
courses.push("React") — courses now has one more item, added at the end. The array variable doesn't need reassignment — push() modifies it directly (mutates it).
.pop()
pop() — remove from the end
Removes and returns the LAST item in the array
The opposite of push(). No arguments needed — pop() always removes whatever is currently last. Useful for "undo last action" type features.
.splice(1,1)
splice(start, count) — remove from anywhere
Removes items starting at a specific index — not just the last one
splice(1, 1) means: start at index 1, remove 1 item. splice(0, 2) would remove the first 2 items. This is how we'll delete a SPECIFIC student from the dashboard later today.
.length
length — count of items
A property (not a method) that tells you how many items are in the array
No parentheses: courses.length not courses.length(). This is how we'll build the "Total Students" counter on the dashboard later today.
let x = {}
{ } — object with key:value pairs
Curly braces create an object — data with labelled properties
Unlike an array (ordered list), an object stores LABELLED data: { name: "Rahul", course: "MERN" }. Order doesn't matter in objects — you access by name, not position.
student.name
. (dot notation) — access a property
Reaches into the object and retrieves the value of a specific key
student.name = "go into student, find the name key, give me its value." This is the most common way to read object data in JavaScript, React, and APIs you'll build later in this course.

Connect the Registration Form
Students already have a Registration Form — now we wire it to memory
Step 1 — create the students array at the top of script.js
frontend/js/script.js — top of file
1let students = []; // holds every registered student
step 2 — inside your form's submit function
inside registerStudent()
// name, email, course already validated
// (from Day 8's form validation)
let student = {
name:name,
email:email,
course:course
};
students.push(student);
console.log(students);
what just happened
Step 1 → Form values (name, email, course) get packed into one object.

Step 2 → That object gets pushed into the students array.

Every new registration adds one more object to the array — without ever overwriting the previous ones.

This is real data storage. Every time someone registers, their info is captured as a structured object and saved into your array. This array — sitting in browser memory — is the foundation for everything we build in the next 90 minutes: the dashboard table, delete buttons, and student counter.
Flow
Form → Object → Array → Memory
Students now understand the complete data journey — visualised step by step
1
Registration Form
User types Name, Email, Course into Bootstrap input fields and clicks Submit. This is raw input — just text in HTML form fields.
2
↓ Object
JavaScript reads each field's value and packs them together into ONE structured object: {name, email, course}. Now the three separate pieces of data are connected and labelled.
3
↓ Array
That object gets push()'d into the students array. The array now holds this student alongside every other student who has registered before.
4
↓ Memory
The data lives in the browser's memory (RAM) for as long as the page stays open. Refreshing the page clears it — that's expected for now. (We'll connect a real database, MongoDB, much later in the course.)
Why this matters: Every real-world web app — Instagram, Amazon, SkillHub's future production version — follows this exact same pattern: collect input → structure it → store it → display it. You just learned the core pattern of full-stack development.

@media queries are the answer to all of these. We write rules that say "only apply this CSS when the screen is below 768px" — and the browser handles the rest automatically.
Concept
What Is a Media Query?
A conditional CSS rule — "apply this style only when a condition is true"
Definition
"If the screen size changes,
apply different CSS."
the analogy
Desktop (≥768px)
Apply your normal CSS — big fonts, side-by-side cards, horizontal nav
↓
Mobile (<768px) → @media fires
Override with mobile CSS — smaller fonts, stacked cards, column nav
how it works in code
style.css — structure
/* ── Normal CSS (desktop) ── */
.hero h2 { font-size:42px; }
/* ── Mobile override ── */
@media(max-width:768px) {
.hero h2 { font-size:28px; }
}
When the screen is ≥768px: browser uses 42px. When screen drops below 768px: browser switches to 28px. Automatically. No JavaScript needed.
Syntax
@media Syntax — Every Part Explained
Click each row below to expand what that part means
@media /* 1. The at-rule */
(max-width: 768px) { /* 2. The condition */
.hero h2 { /* 3. The selector */
font-size: 28px; /* 4. The override */
}
} /* 5. Close the @media block */
@media
The at-rule — starts a conditional CSS block
Tells the browser: "what follows only applies when a condition is met." The @ symbol marks it as a special CSS rule.
CSS has other at-rules too: @keyframes (animations), @import (loading files), @font-face (custom fonts). @media is the most commonly used one. It is always placed at the bottom of style.css, after all your normal CSS.
max-width:768px
The condition — "screen width is 768px or less"
max-width: 768px means "trigger when the viewport is AT MOST 768px wide." This covers all phones and most portrait tablets.
768px is the industry standard mobile breakpoint. Common breakpoints: 480px (small phones), 768px (tablets/phones), 1024px (tablets/small laptops), 1200px (wide desktop). You can use any value that makes sense for your layout.
.hero h2 { … }
The selector + override — same CSS, new value
You rewrite the same selector that you used in normal CSS — but with a new value that suits smaller screens.
You do NOT need to repeat all the properties — only the ones you want to change. The browser uses the original CSS for anything not mentioned inside the @media block. This is called "overriding" — the @media value wins when the condition is true.
} ← outer brace
Close the @media block — easy to forget!
Every @media block needs its own closing brace in addition to the selector's closing brace. That's two closing braces total at the end.
Forgetting the outer closing brace is the #1 @media bug for beginners. If your media query isn't working, check that you have TWO closing braces — one for the selector and one for the @media block. VS Code will highlight mismatched braces.
Always write @media at the BOTTOM of style.css — after all your regular CSS. This ensures mobile overrides always win over desktop rules, since CSS reads top to bottom.

Why does width:100% make cards stack?
The course container still has display:flex; flex-wrap:wrap. When each card is forced to width:100%, it takes up the full row width — so the next card has no room and must wrap to the next line. Result: one card per row = vertical stacking.

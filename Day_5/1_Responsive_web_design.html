What Is Responsive Design?
The definition, the why, and the three techniques we'll use
Responsive Design means a website automatically adjusts itself according to the screen size.
60% Mobile Traffic
Over 60% of global web traffic comes from mobile devices. A non-responsive site loses most of its users.
Google Ranks Mobile First
Google uses "mobile-first indexing" — non-responsive sites rank lower in search results.
Job Requirement
Every real-world MERN project must be responsive. It's a baseline expectation in any job interview.

Browser uses device width → readable, responsive layout
width=device-width
width=device-width
Sets the viewport width to match the physical screen width of the device
Without this, browsers pretend the screen is 980px wide (desktop default) and shrink everything to fit. A phone is only 375px wide — so text becomes microscopic. This attribute tells the browser to use the device's actual width instead.
initial-scale=1.0
initial-scale=1.0
Sets the initial zoom level to 100% — no zoom-in or zoom-out on first load
1.0 = no zoom (100%). 2.0 = 200% zoom (everything appears double size). Without initial-scale, some browsers zoom in or out by default, breaking your carefully designed layout. Always set it to 1.0.

Explain
width:100% vs height:auto
Click each property to expand — the most important responsive CSS pair
width: 100%
width: 100% — take full width of parent
Makes the image stretch to fill 100% of its parent container's width
100% is relative — it always means "100% of the parent element." So if the parent is 375px (phone), the image is 375px. If the parent is 1200px (desktop), the image is 1200px. This is why it's "responsive" — it adapts to whatever container it lives in. Never overflows. Always fits.
height: auto
height: auto — maintain aspect ratio
Lets the browser calculate the height automatically based on the image's original proportions
If you set width:100% but leave height fixed (e.g. height:200px), the image gets squished or stretched as the width changes. height:auto says "calculate the correct height to keep the original width:height ratio." A 1920×1080 image (16:9 ratio) will always stay 16:9, no matter how wide it is displayed.
max-width: 100%
max-width: 100% — the safer alternative
Image grows up to its natural size, but never overflows the container
width:100% forces the image to always stretch to full container width — even if the image is smaller than the container (causing blurry upscaling). max-width:100% is gentler: the image stays its natural size on large screens but shrinks on small screens. Use max-width for most images, width:100% for hero banners.
display: block
display: block — remove the bottom gap
Images are inline by default — this creates a small gap below them
Browsers treat <img> as inline elements (like text), which adds a small gap at the bottom (the "descender" space for letters like p, y, g). Adding display:block removes this gap. When width:100% and display:block are combined, images behave perfectly in responsive layouts.
complete responsive image rule — best practice
style.css
.hero-image {
width: 100%; /* fills the container */
height: auto; /* keeps aspect ratio */
display: block; /* removes bottom gap */
}

Mobile-first approach: Professional developers write CSS for mobile screens first, then add media queries for larger screens. This is called "mobile-first" and is the industry standard approach we'll use in Hour 2.
Common breakpoints: 480px (small phone) · 768px (tablet/large phone) · 1024px (small laptop) · 1280px (desktop). We'll use 768px as the main breakpoint for SkillHub.

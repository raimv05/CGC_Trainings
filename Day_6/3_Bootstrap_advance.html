the answer
Bootstrap's JS bundle handles it for you.

This is exactly why Bootstrap is used by millions of developers. The hamburger toggle, dropdowns, modals — all pre-written in Bootstrap's JS. You just add the class names to your HTML and it works. This is the power of a framework.

Deep Dive
Every Navbar Class — Click to Explore
Each class name is doing something specific — click a row to find out what
navbar
Base Component Class
Tells Bootstrap "this element is a navbar." Applies the base flex layout, padding, and display properties automatically.
Every Bootstrap component has a "base class" that must be present: navbar, btn, card, modal. Without it, the component-specific modifier classes don't work.
navbar-expand-lg
Breakpoint for Hamburger Toggle
Links are visible as a row on screens ≥992px (lg). Below 992px, links collapse and the ☰ toggler button appears.
Other options: navbar-expand-sm (576px), navbar-expand-md (768px), navbar-expand-xl (1200px). Choose based on where your nav links become too cramped.
navbar-dark
Light Text for Dark Background
Sets link colours to white/light so they're readable on a dark background. Use navbar-light for white/grey backgrounds.
navbar-dark / navbar-light only sets text colours — you still need a separate bg-* class (bg-dark, bg-primary) to set the background colour.
bg-dark
Dark Background Colour
Applies Bootstrap's dark background (#212529). Can be swapped for bg-primary (blue), bg-light (white/grey), bg-success (green), etc.
Bootstrap's bg-* utilities match the btn-* colours exactly. bg-primary = #0d6efd, bg-success = #198754, bg-danger = #dc3545.
navbar-toggler
The ☰ Hamburger Button
Styled button that only appears on mobile. The data-bs-toggle and data-bs-target attributes tell Bootstrap's JS what to expand/collapse when clicked.
data-bs-target="#menu" must match the id of the collapse div below it — that's how Bootstrap knows which element to show/hide.
navbar-nav ms-auto
Nav List + Push to Right
navbar-nav styles the <ul> as a horizontal flex list. ms-auto (margin-start: auto) pushes the list to the right side of the navbar.
ms = margin-start (left in LTR layouts). me = margin-end (right). Bootstrap 5 replaced ml-auto (Bootstrap 4) with ms-auto.
collapse navbar-collapse
Collapsible Menu Container
The div that wraps the nav links. On mobile, Bootstrap hides this div. When ☰ is clicked, Bootstrap toggles the show class on it to reveal the links.
The id="menu" on this div is referenced by data-bs-target="#menu" on the toggler. If these don't match, the hamburger won't work.

p-5 content

mt-5
Margin Top
CSS equivalent: margin-top: 3rem (48px)
Prefix t = top · b = bottom · s = start(left) · e = end(right) · x = left+right · y = top+bottom

Custom CSS vs Bootstrap — Side by Side
See exactly how many lines Bootstrap saves you
❌ custom CSS way (~22 lines)
style.css + index.html
/* CSS file */
nav { background:#1e293b; display:flex;
justify-content:space-between;
padding:14px 24px; }
.btn-custom { background:#2563eb;
color:white;
padding:8px 20px;
border:none;
border-radius:6px;
cursor:pointer; }
.btn-custom:hover { background:#1d4ed8; }
.card { background:white;
border-radius:8px;
box-shadow:0 2px 8px rgba(0,0,0,.1);
padding:20px; }
/* + write media queries by hand */
/* + write hover states by hand */
/* + write hamburger JS by hand */
≈ 22 lines CSS + manual JS for hamburger
✅ bootstrap way (~3 lines HTML)
index.html only
<!-- navbar: dark, responsive, auto hamburger -->
<nav class="navbar navbar-expand-lg navbar-dark bg-dark">
<!-- ... navbar content ... -->
</nav>
<!-- button with hover state built in -->
<button class="btn btn-primary">Join Now</button>
<!-- card: shadow, border-radius, padding -->
<div class="card shadow p-3">
<!-- card content -->
</div>
<!-- zero extra CSS needed -->
<!-- responsive built in -->
<!-- hamburger menu built in -->
≈ 3 lines HTML · zero extra CSS · hamburger free
This is what a CSS framework does — it pre-writes thousands of lines of production-tested CSS so you never have to. What took an experienced developer hours to write custom is now a class name.

container
Centers content + adds max-width
Wraps everything in a centered block. Adds automatic left/right padding.
Without .container, your grid rows stretch to the full window width. .container applies a responsive max-width and auto margins to keep content readable on widescreen monitors.
row
Creates one horizontal "shelf"
Each .row is a flex container that holds its .col children side by side.
.row applies display:flex and negative horizontal margins to offset the column gutters. Every group of columns that should sit on the same line needs its own .row.
col
Auto-equal column — splits space evenly
Plain .col divides the row equally between all .col siblings — no number needed.
Two .col siblings → each gets 50%. Three .col siblings → each gets 33.3%. This is the quickest way to create equal columns without thinking about numbers.

Breakpoints — sm / md / lg / xl
The prefix in col-md-4 tells Bootstrap when to switch from columns to stacked
breakpoint reference table
Class prefixApplies when screen is…Typical deviceExample use
col- (no prefix) All screen sizes — always columns All Icons that always sit side by side
col-sm-* ≥ 576px Large phones, small tablets Social media cards
col-md-* ≥ 768px Tablets, laptops ← We use this Course cards, product grids
col-lg-* ≥ 992px Laptops, desktops Dashboard side-by-side panels
col-xl-* ≥ 1200px Wide desktop monitors Admin panels, data tables

card
Outer wrapper — adds border + shadow
Gives the element a white background, 1px border, border-radius, and subtle shadow.
Bootstrap's .card replaces ~8 CSS lines you wrote manually in Days 3–4 (border-radius, box-shadow, background, overflow). One class name does it all.
card-body
Inner padding — gives content breathing room
Adds 1rem (16px) padding on all sides inside the card.
Always nest card-body directly inside card. It's the padding container. Without it, text and buttons would touch the card's edges.
btn btn-primary
Blue Bootstrap button — two classes needed
btn applies base button styles; btn-primary sets the blue (#0d6efd) colour.
Other colour variants: btn-success (green), btn-danger (red), btn-outline-primary (bordered). Both classes are required — btn alone gives no colour.

row g-4 — the g-4 adds a gutter (gap) between columns. g-4 = 1.5rem gap. Without it, columns sit right next to each other with no breathing room.

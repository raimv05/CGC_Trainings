Question
The Tea Recipe Question
Every morning your mother makes tea. Does she explain the recipe every single day?
Does your mother explain the entire tea recipe every single morning before making it?
No!
The recipe is written once, in her memory.
Every morning, she just "calls" the recipe — without re-explaining it.
This is exactly how a function works in JavaScript.
📝
Write the recipe once
Boil water → Add tea leaves → Add milk → Add sugar → Strain
🫖
Call it every day
"Make tea" — no need to re-explain the steps each time
☕
Same result, every time
Consistent, repeatable, no wasted effort
This IS a function: A reusable block of code — write the instructions once, then "call" them by name whenever you need that exact behaviour again.

console.log vs return — the key difference
console.log() — just displays a value in the console. Cannot be reused elsewhere.
return — gives back a value that can be stored in a variable and used anywhere else in your code.
A function can return data. This is one of the most powerful ideas in programming — you'll use return constantly in React components and Node.js APIs later in this course.
return stops the function: any code written after return inside the same function never runs. Always the last line that matters in a function.
Explain
Line-by-Line Explainer
Click any row to expand — every function concept from today
function
function — keyword
Tells JavaScript "I'm about to define a reusable block of code"
This is the classic way to define a function. Modern JS also has arrow functions (const greet = () => {}) which you'll see in React. For now, the function keyword is the clearest way to learn the concept.
greet
Function name
The label you use to call this function later
Use camelCase, descriptive names: greetStudent, calculateFees, showCourse. A good function name tells you what it does without reading the code inside. Avoid vague names like doStuff or fn1.
(name)
Parameters — the parentheses
Placeholders for data the function needs to do its job
Empty () = no data needed. (name) = needs one piece of data. (name, course) = needs two, separated by commas. Parameters only exist INSIDE the function — you can't use name outside it.
{ … }
Function body — curly braces
Everything between { and } is the actual code that runs when the function is called
This is the "recipe" — the steps. Code here only executes when the function is called, not when it's defined. Indent the content inside for readability (we use 4 spaces or 1 tab, consistently).
greet();
Function call — the parentheses trigger it
Writing the function name followed by () actually RUNS the code inside
Without (), just writing greet refers to the function itself, but doesn't run it. The parentheses are what trigger execution. This is the single most common beginner mistake — forgetting the parentheses when calling a function.
"Rahul"
Argument — the actual value passed in
The real data you send when calling the function
Parameter (in the definition) vs Argument (in the call) — related terms, technically different. function greetStudent(name) → name is the parameter. greetStudent("Rahul") → "Rahul" is the argument.
return a + b;
return statement
Sends a value back out of the function, to be used elsewhere
Without return, a function call evaluates to undefined — you can't store its result in a variable usefully. With return, you CAN write let total = add(10, 20) and total becomes the returned value (30).

15 >= 18 is false. The code inside { } is simply skipped — nothing runs, nothing prints. The console stays empty. This is the key behaviour of a plain if with no else.

why else matters
if(…) {…}
Runs ONLY when condition is true
If false, this entire block is skipped completely.
This is exactly the same if block from the previous slide — nothing new here.
else {…}
Runs ONLY when the if condition is false
else has no condition of its own — it's the "anything that didn't match the if" branch.
A program always runs EITHER the if block OR the else block — never both, never neither. This guarantees you always handle both outcomes of a yes/no decision.
This is real validation logic. Every form, every login system, every age check on the internet uses exactly this if/else pattern under the hood.
Code
else if — More Than Two Outcomes
A grading system needs more than yes/no — chain multiple conditions together
Type this grading logic into script.js
frontend/js/script.js
1let marks = 85;
2
3if(marks >= 90) {
4console.log("A Grade");
5} else if(marks >= 75) {
6console.log("B Grade");
7} else {
8console.log("C Grade");
9}
how javascript checks this, top to bottom
1
marks = 85. Check marks >= 90 first
Is 85 ≥ 90? No — false. Skip this block, move to the next check.
2
Check marks >= 75
Is 85 ≥ 75? Yes — true! Run this block: "B Grade". Stop checking further conditions.
3
else block is skipped entirely
Once a match is found, JavaScript never checks the remaining conditions — even the final else.
Console
▶ B Grade

Order matters in else-if chains. Always write the highest/most-specific condition FIRST. If you checked >=75 before >=90, a mark of 95 would incorrectly stop at "B Grade".
Reference
Comparison Operators
The building blocks of every condition you write
==
Loose equal
===
Strict equal
!=
Not equal
>
Greater than
<
Less than
>=
Greater or equal
<=
Less or equal
OperatorMeaningExampleResult
== Equal value (ignores data type) 5 == "5" true
=== Equal value AND equal type — always prefer this 5 === "5" false
!= Not equal age != 18 depends on age
> Strictly greater than marks > 90 depends on marks
>= Greater than or equal to age >= 18 true if 18 or older
Use ===, not ==, in your code. == converts types before comparing (causing confusing bugs like 0 == false being true). === checks value AND type together — predictable and safe. This is a strict industry rule.
Reference
Logical Operators — Combine Conditions
&& (AND), || (OR), ! (NOT) — chain multiple checks into one condition
&&
AND
Both sides must be true
||
OR
At least one side must be true
!
NOT
Flips true ↔ false
&& example — registration needs BOTH conditions
let age = 20;
let hasEmail = true;
if(age >= 18 && hasEmail) {
console.log("Can register");
}
Needs to be 18+ AND have an email. If either is false, registration fails.
|| example — accept EITHER login method
let hasGoogle = false;
let hasEmail = true;
if(hasGoogle || hasEmail) {
console.log("Login allowed");
}
Just ONE needs to be true. Even though hasGoogle is false, hasEmail being true makes the whole condition true.
! (not) — flips the value
let isLoggedIn = false;
if(!isLoggedIn) {
console.log("Please log in"); // runs, because !false = true
}

Project Task
Registration Form — Add Validation Logic
Use everything from today — if/else, comparison & logical operators — to validate real input
Add to script.js — runs when the form is submitted
frontend/js/script.js
1function validateForm() {
2
3let name = document.getElementById("name").value;
4let age = document.getElementById("age").value;
5
6if(name === "") {
7alert("Name is required");
8return; // stop here — don't continue
9}
10
11if(age < 18) {
12alert("You must be 18 or older");
13return;
14}
15
16alert("Registration successful, " + name);
17}
line-by-line — click to expand
name === ""
Check for an empty string
If the user submitted without typing a name, .value will be exactly "" (empty).
We use === (strict equal) here, not ==, following the rule from earlier in this hour. Always compare with === for predictable results.
return;
Exit the function immediately
return with no value just stops the function — none of the code after it runs.
This is critical for validation: if the name is empty, we show the alert and STOP — we never want to reach "Registration successful" with bad data.
age < 18
Reject under-18 registrations
If age (as typed by the user) is below 18, block registration with an alert.
Note: .value from an input always returns a String, even for type="number" inputs. In real production code we'd convert it with Number(age) before comparing — we'll cover that conversion in a later lesson.
html — connect the submit event

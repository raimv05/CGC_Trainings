What happened? display:flex turns the parent (.course-container) into a Flex Container. Its direct children automatically become Flex Items and arrange in a row. No other CSS needed for this effect.
Two new vocabulary words — remember them:
Flex Container
The parent element with display:flex on it. Controls how children are arranged.
Flex Items
The direct children inside the container. They respond to the container's flex rules.

Students love this moment. Ask them to resize the browser window after adding flex-wrap. Cards automatically rearrange — no media queries needed. This is their first taste of responsive design.
How does wrap decide when to break? It wraps when items exceed the container width. Since each card has a fixed width: 260px, the browser calculates how many fit per row and wraps the rest automatically.

Key idea: The .course-container div is the Flex Container. Each .card inside it becomes a Flex Item. We apply display:flex to the parent, never the cards themselves.

display:flex
Makes .course-container a Flex Container
Children (cards) immediately line up in a row.
Without this line, every other property in the block is meaningless. This is the on/off switch for Flexbox.
justify-content:center
Centres the cards horizontally
If there are only 2 cards, they sit in the middle — not left-aligned.
Try space-between to push cards to edges. Try flex-start for left-alignment. Pick what suits your design.
gap:25px
25px space between every card
No gap before the first card, no gap after the last — just between.
Using margin on cards instead would add space outside the container too. gap is always cleaner inside flex.
transform:translateY(-10px)
Card lifts up on hover
The transition:.3s on .card makes this animate smoothly.
translateY moves the element along the Y-axis (vertical). Negative = up. Combined with box-shadow, this creates a "lifting card" effect popular on modern sites.

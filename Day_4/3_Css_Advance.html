key points to explain
<section>
Semantic Sectioning
Use <section> for distinct page regions — better than <div> for accessibility and SEO.
Screen readers announce section boundaries. Google uses sections to understand page structure.
class="features"
Section Container
Wraps the whole section so we can add padding, background, and text-align to the entire block.
Think of it as the "room" — everything inside inherits its background colour and padding.
class="feature-container"
Flex Parent
This div becomes the Flex Container. CSS will set display:flex on it to arrange the three cards side-by-side.
Exactly the same pattern as .course-container from Hour 2!
class="feature"
Flex Child (Card)
Each .feature div is one card — it gets its own width, padding, background, and hover effect in CSS.
Three siblings → three cards side by side when the parent has display:flex.
Pattern recognition: Notice the container → cards pattern is identical to .course-container → .course-card. This is the most common Flexbox pattern on the web.

what each new property does
background: #eef4ff
Section Background
A very light blue — visually separates this section from the white courses area above it.
Alternating light backgrounds (white → light blue → white) is a standard landing-page technique to guide the eye.
width: 300px
Fixed Card Width
Sets each feature card to exactly 300px. Combined with flex-wrap, cards drop to new lines on smaller screens.
Could also use flex: 1 1 280px for a more fluid layout.
transition: .3s
Smooth Animation
Makes the scale() hover animate smoothly over 0.3 seconds instead of jumping instantly.
Full form: transition: all 0.3s ease. The shorthand .3s applies to all animatable properties.
transform: scale(1.05)
Scale on Hover ⭐
Enlarges the card to 105% of its size on hover. 1 = 100%, 1.05 = 105%. Does NOT affect surrounding layout.
Unlike width/margin changes, transform:scale() doesn't shift other elements — the card "floats" larger while neighbours stay put.
Always pair transform with transition. Without transition, the hover snaps instantly and looks glitchy. The .3s is the magic that makes it feel polished.

what's happening
Understanding scale values
ValueResultUse when…
scale(1)No changeDefault state
scale(1.05)+5% biggerSubtle, polished
scale(1.1)+10% biggerMore noticeable
scale(0.95)Shrinks 5%Press/click effect
1.05 is the industry standard — large enough to feel interactive, small enough to feel professional. Anything above 1.1 can feel like a toy.
transition: .3s is the invisible hero here. Remove it and hover snaps instantly — add it and the card smoothly grows. Always add transition when using transform.

Deep Dive
CSS Properties — Click to Explore
Every property used in Hour 3 — click a row to see the detail
padding: 60px
Section Breathing Room
60px of space on all four sides makes the section feel spacious, not cramped. Sections need more padding than individual cards.
Rule of thumb: cards use 15–25px padding, sections use 40–80px padding. The difference in scale signals hierarchy.
text-align: center
Centre Alignment
Applied to .features, it centres the h2 heading and is inherited by child elements unless overridden.
Note: text-align:center on a flex container does NOT centre flex items — you still need justify-content:center for that.
justify-content: center
Flex Main Axis Alignment
Centres the feature cards horizontally within the .feature-container flex row. Same property you learned in Hour 2!
If you had 4 cards and wanted the first and last touching the edges, you'd use space-between instead.
transform: scale(1.05)
Scale Transform ⭐ New!
Grows the element to 105% of its original size. Crucially — does NOT shift surrounding elements unlike changing width/margin.
transform happens on the GPU (graphics card), making it the smoothest way to animate. Always prefer transform over animating width, height, or margin.
transition: .3s
Animation Duration ⭐ New!
Shorthand for transition: all 0.3s ease. Animates ALL property changes over 0.3 seconds with an ease timing curve.
The ease curve starts fast and slows at the end — it feels natural. The full property is: transition: property duration timing-function delay.
box-shadow: 0 0 15px rgba()
Diffused Shadow
0 0 = no X or Y offset = shadow spreads equally in all directions. 15px blur = soft, glowing shadow. Great for floating card effect.
Contrast with course cards (Hour 2) which used 0 2px 6px — an offset shadow that simulates light from above. This is a design choice.
Always use transform + transition for hover effects — never animate width, height, margin, or padding directly. Those cause the browser to recalculate the entire layout (reflow) on every frame, which is slow and janky.

Concept
ul vs ol vs li — Explained
Know when to use each — the rule is simpler than you think
<ul> — Unordered List
<ul>
<li>HTML & CSS</li>
<li>JavaScript</li>
<li>React.js</li>
</ul>
  • HTML & CSS
  • JavaScript
  • React.js
• Bullet points — order doesn't matter
Use for: skills, features, course lists, navigation items
<ol> — Ordered List
<ol>
<li>Install VS Code</li>
<li>Install Node.js</li>
<li>Create Project</li>
</ol>
  1. Install VS Code
  2. Install Node.js
  3. Create Project
1. 2. 3. — order matters
Use for: steps, rankings, instructions, top-10 lists
quick decision rule
Ask: "Would changing the order change the meaning?"
No → Use <ul> (courses, skills, features)
Ask: "Would changing the order change the meaning?"
Yes → Use <ol> (setup steps, instructions)
<li> must always be inside <ul> or <ol> — never write a standalone <li>. Think of it like a seat (li) that must be inside a bus (ul/ol). A seat on its own makes no sense.

Line-by-Line — New Sections

<section>
Second <section> — About Us
A new semantic content block added after the Hero section
A page can have many <section> elements — one per logical content block. Each should have its own <h2>. We're building the page layer by layer: Hero → About → Courses → more sections in future hours.
<h2>About…</h2>
Section Heading — h2
Sub-heading for this section (h1 is already used in <header>)
Heading hierarchy matters: h1 (one per page) → h2 (section titles) → h3 (sub-sections). Google uses this hierarchy to understand the importance and structure of your content. Never skip levels.
<p>SkillHub…</p>
Paragraph
Body text describing the About section
Text split across multiple lines in the code still renders as one paragraph in the browser. HTML ignores extra whitespace. The <p> tag itself creates the spacing — not the line breaks in your code.
<section>
Third <section> — Our Courses
Another semantic block, this time for the course listing
This is the "Courses" section. It's separate from About because it's a different topic. Keeping them in separate <section> tags makes it easy to style them differently with CSS later.
<ul>
Unordered List
Creates a bullet-point list — order of courses doesn't matter
<ul> is a container — it holds <li> items. By itself it renders nothing. You must put <li> tags inside it. The bullet style (•) can be changed or removed later with CSS.
<li>React.js</li>
List Item
One item in the list — each course gets its own <li>
Notice: <li> has both an opening AND closing tag — don't forget </li>. Inside <li> you can put text, images, links, even nested lists. For now, plain text is perfect.
</ul>
Closing </ul>
Ends the unordered list
Common mistake: forgetting the closing </ul> or closing it too early. All <li> items must be between <ul> and </ul>. In VS Code, the closing tag is usually suggested automatically.

Tags
The Tags We'll Use in This Hour
Six new tags to build the Hero Section content
<h1>
Most Important Heading — Page Title
The biggest, boldest heading. Use it once per page for the main message. On SkillHub it is already used for "SkillHub LMS" in the <header> from Hour 1. Google gives it the most SEO weight.
<h1>SkillHub LMS</h1> — already written in Hour 1
<h2>
Section Heading — Our Hero Title
Because <h1> is taken, the hero heading uses <h2>. Can be used many times on a page — once per section. Always comes after h1 in hierarchy.
<h2>Become a Full Stack Developer</h2>
<h3>
Sub-Heading — Inside Sections
Third-level heading. Smaller than h2. Used inside a section to label cards or sub-topics — for example, individual course names in the courses list.
<h3>MERN Stack Course</h3>
<p>
Paragraph — Body Text / Description
Regular body text. The browser adds a gap above and below each <p> automatically. Use it for the hero description — keep it to 1–2 sentences.
<p>Learn HTML, CSS, JavaScript, React, Node.js and MongoDB.</p>
<button>
Button — Call To Action (CTA)
Creates a clickable button. The CTA tells the visitor what to do: "Explore Courses," "Join Now," "Start Learning." Without CSS it renders as the browser's plain grey button — we style it in Day 3.
<button>Explore Courses</button>
<br>
Line Break — Force a New Line
Inserts a line break inside text. Self-closing — no closing tag. Useful inside a heading to break at a specific word. Don't use for spacing between elements — use <p> tags or CSS margin instead.
Learn HTML, CSS <br> and JavaScript.

Heading Hierarchy
h1 is the biggest, h3 the smallest we use today — always go in order
<h1>
SkillHub LMS

<h2>

<h4>
Module 1: HTML Basics
Rarely needed
Wrong — skip levels
bad example
1<h1>SkillHub LMS</h1>
2<h4>Hero Text</h4> <!-- skipped h2,h3! -->
3<h1>Courses</h1> <!-- second h1! -->
❌ Never skip levels. Never use two <h1> tags on one page.
Correct — stay in order
good example
1<h1>SkillHub LMS</h1> <!-- header -->
2<h2>Become a Full Stack...</h2>
3<h3>MERN Stack Course</h3>
✅ h1 → h2 → h3 — always in sequence, like a document outline.
Think of it like an outline: h1 = book title, h2 = chapter title, h3 = section inside a chapter. Screen readers and Google both follow this hierarchy to understand your page structure. Breaking it confuses both!

Explain
Tag-by-Tag Explanation
Click any row to expand why that tag is used in the hero
<h2>
Heading 2 — Hero Title
The main visible headline of the hero section
We use <h2> here (not <h1>) because <h1> is already used for "SkillHub LMS" in the <header> from Hour 1. There must be only one <h1> per page. <h2> is the correct next level for the hero section heading. Google reads this text as a key phrase for the page topic.
<p>
Paragraph — Hero Description
Body text explaining what SkillHub is about
The <p> tag tells the browser "this is a block of text." It automatically adds space above and below. You can have multiple <p> tags. This is where you write the hook — what does the visitor get? For a hero, keep it to 1–2 sentences. More than that and visitors stop reading.
<button>
Button — Call To Action (CTA)
The action you want the visitor to take
The CTA button is the most important element after the heading. The text must be specific and action-oriented — "Explore Courses" is better than "Click Here." Without CSS it renders with the browser's default grey button style. In Day 3 we add colour, border-radius, padding, and hover effects to make it look great.
<br>
Line Break — Force New Line
Breaks the line without starting a new paragraph
Unlike <p> which creates a paragraph gap, <br> only breaks the line — no extra spacing. Self-closing — no </br> needed. Useful inside a heading to control where text wraps: <h2>Full Stack <br> Developer</h2>. Don't use <br> to add spacing between elements — use <p> or CSS margin for that.
<section>
Section (from Hour 1) — The Container
The semantic wrapper for the entire hero block
From Hour 1 — <section> tells the browser this is a thematically grouped block. All our hero content (h2, p, button) lives inside this <section>. In Hour 3+ we will add more sections for "Courses," "Testimonials," etc. Each major block of the page gets its own <section>.
Connecting to Hour 1: The semantic structure from Hour 1 (<header>, <nav>, <section>, <footer>) is still exactly there. We only added content inside the <section>. The skeleton we built in Hour 1 is unchanged — good code is built in layers.

Introducing <img> — Add Images to HTML
A special self-closing tag — all information lives in its attributes
the img tag syntax
image syntax
1<img
2src="https://picsum.photos/400"
3alt="Course Image"
4width="400"
5height="250"
6>
7<!-- No closing tag needed! -->
Self-Closing — like <br>
<img> has no closing tag. It has no content between tags — all information is in attributes. Same rule as <br> from earlier in this hour.
rendered result
index.html
src="https://picsum.photos/seed/skillhub/400/200"
picsum.photos — free placeholder images
We use https://picsum.photos/400/200 as a placeholder — it returns a random photo at exactly that size. Change the numbers to change dimensions. Change seed/name to get a consistent image.
the four attributes — every student must know these
src = "..."
Source — the URL or file path of the image. Required. Without it nothing shows on the page.
src="images/hero.jpg" or src="https://..."
alt = "..."
Alternative text — shown if image fails to load; read by screen readers. Always include for accessibility.
alt="Students learning web development"
width = "..."
Width in pixels. Controls how wide the image displays. Prevents layout shift while the page loads.
width="400" — displays at 400px wide
height = "..."
Height in pixels. Set both width and height together to reserve space and prevent content jumping.
height="250" — displays at 250px tall

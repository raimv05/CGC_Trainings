Concept
What Are Semantic Tags?
Tags that describe meaning, not just appearance
Non-Semantic HTML
old style
1<div id="header">
2<div class="nav"></div>
3</div>
4<div class="content">
5<div class="box"></div>
6</div>
7<div id="footer"></div>
❌ Just <div> tags everywhere. A browser sees "box box box" — no meaning at all.
Semantic HTML
modern style
1<header>
2<nav></nav>
3</header>
4<section>
5<article></article>
6</section>
7<footer></footer>
✅ Each tag tells everyone — browser, Google, screen reader — exactly what it contains.
semantic = meaning
Semantic comes from the Greek word for meaning. A semantic HTML tag doesn't just create a box — it describes what's inside the box. <header> doesn't just say "here is a div at the top" — it says "this is the page header." That matters for SEO, Accessibility, and Readability.
Tags
The 5 Semantic Tags
Introduce each tag — purpose and real-world use
<header>
Top Area — Site Identity
Contains the logo, site title, and sometimes the navigation. Usually the first thing inside <body>. On SkillHub it holds the "SkillHub LMS" heading.
GoogleYouTubeAmazon— all have a <header>
<nav>
Navigation Links
Wraps the site's main navigation menu. Links to other pages or sections. Screen readers announce this as "navigation" so users can skip directly to it.
<section>
Content Blocks / Grouped Content
Groups related content together. A page can have multiple sections — Hero section, Courses section, Testimonials section. Usually has a heading inside.
<article>
Self-Contained, Independent Content
Content that makes sense on its own — a blog post, a course card, a news article. Could be copy-pasted to another page and still make sense.
<footer>
Bottom Area — Closing Info
Copyright notice, social links, contact info, privacy policy links. Always at the very bottom of the page content.

Explain
Tag-by-Tag Explanation
Click any row to expand the explanation
<header>
Header Element
Top section — logo, site name, top navigation
Can appear more than once (page header AND article header). Google gives it higher weight for SEO. Screen readers allow users to jump past it. Always the first section inside <body> for the main page header.
<nav>
Navigation Element
Site's main navigation links
Tells screen readers "this is a navigation landmark." Users with disabilities can jump directly to it. Should contain your most important page links. Later we'll fill it with <a> anchor tags.
<section>
Section Element
Thematically grouped content — always with a heading
Use <section> when content belongs together under a shared theme. Best practice: always include a <h2> or <h3> inside. Examples: Hero section, Courses section, Testimonials section.
<article>
Article Element
Self-contained, independently distributable content
The test: "Could I copy this to another site and it still makes sense?" If yes → <article>. Use for blog posts, course cards, news items, product cards. Often nested inside <section>.
<footer>
Footer Element
Bottom of page — copyright, links, contact info
Like <header>, can appear inside <article> too (article footer with author info). Main page footer always goes at the very bottom of <body>. Screen readers announce it as "contentinfo" landmark.
<!DOCTYPE html>
DOCTYPE
Tells browser: this is an HTML5 document
Without DOCTYPE, browsers go into "quirks mode" and may render pages inconsistently. Always the very first line of every HTML file. Not a tag — it's a declaration.
<html>
HTML Root Element
The container wrapping the entire page
Everything lives inside <html>. Add lang="en" for English pages — it helps screen readers use the correct pronunciation and helps search engines index the language correctly.
<head>
Head Element
Invisible metadata — title, CSS links, meta tags
Nothing inside <head> is shown on the page. It holds the browser tab title, charset, viewport settings, CSS file links, and page description for Google.

Why
Why Semantic HTML Matters
Three real reasons — not just "best practice"
SEO
Google's crawlers read semantic tags to understand what your page is about. A proper <header> and <h1> help rank your page higher in search results.
Accessibility
Screen readers (used by visually impaired users) announce landmarks — "header," "navigation," "main." Semantic HTML makes sites usable for everyone.
Readability
A team reading <article> instantly knows it's a post. Reading <div class="box7">? Nobody knows. Semantic HTML is self-documenting code.
real-world impact
Without semantics: "I see a box at the top, a box of text, another box at the bottom."
With semantics: "I see a site header, a navigation bar, a main section, a footer."
Industry standard: In a real job, your code reviewer will ask you to replace meaningless <div> tags with semantic equivalents. Learning this now puts you ahead.

Reference
Semantic Tags Reference
Keep this open while you code — complete at-a-glance guide
Tag Purpose Where on Page Real Example
<header> Site identity — logo, title, top nav Top of <body> SkillHub LMS logo + title
<nav> Main navigation links Inside <header> or below it Home · Courses · About · Contact
<main> Primary page content (unique to this page) Between <nav> and <footer> Course listing page content
<section> Thematically grouped content Inside <main> Hero section, Courses section
<article> Self-contained independent content Inside <section> Individual blog post, course card
<aside> Sidebar — loosely related content Beside <section> Related courses, ads, author bio
<footer> Bottom info — copyright, links Bottom of <body> Copyright 2026 · Privacy Policy
Rule of thumb: If you're tempted to write <div class="header">, <div class="nav">, or <div class="footer"> — use the semantic tag instead.
Note: <aside> and <main> are bonus tags — we'll add them to SkillHub in a later session when we build the full layout.
